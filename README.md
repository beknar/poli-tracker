# poli-tracker

**Track members of Congress' stock purchases — and how those bets have performed.**

poli-tracker is a small Flask + SQLite web app that collects the stock
**purchases** disclosed by U.S. House and Senate members over the last 6 months,
caches them locally, and presents them in a professional dashboard. For every
holding it shows the price at purchase versus today, the estimated value then and
now, and a one-click performance graph from the purchase date to the present.

> Educational / sample project — **not investment advice**. Trade data comes from
> public STOCK Act disclosures; dollar values are estimates (see *Notes* below).

## Features

- **Backend (Python):** pulls recent *purchase* disclosures for the House and
  Senate, prices each holding (at purchase vs. now), and stores everything in
  **SQLite**. The first run builds the database; later runs **upsert** new
  purchases and refresh current prices.
- **Dashboard (web):** a Bootstrap + DataTables list with **one row per member** —
  `Member (link) | Chamber | Purchases | Est. value now`.
- **Member detail page:** clicking a member opens their last-6-months purchases —
  total cash disclosed, original $/share, current $/share, % change, estimated
  value now, and a link to a performance chart per holding.
- **Performance graphs:** each priced holding has a **📈 graph** (Chart.js) of the
  stock from the purchase date to today.
- **HTTPS:** serve behind nginx with a self-signed certificate — see
  [`deploy/README.md`](deploy/README.md).

## Data sources (free, no API key)

- **Trades:** the public House/Senate "stock watcher" datasets — community mirrors
  of the official disclosures (see `backend/config.py` for the URLs).
- **Prices:** [`yfinance`](https://pypi.org/project/yfinance/) (Yahoo Finance).

## Architecture

```
poli-tracker/
├── app.py                  # Flask: dashboard, /refresh, /api/price-history
├── backend/
│   ├── config.py           # lookback window, data-source URLs, DB path (env-overridable)
│   ├── db.py               # SQLite schema, upserts, price cache, dashboard queries
│   ├── congress.py         # fetch + normalize disclosure data (purchases, last 6 mo)
│   ├── prices.py           # yfinance lookups (historical / current / series), cached
│   └── ingest.py           # orchestrates a refresh (first run + incremental updates)
├── templates/dashboard.html
├── static/{app.js,style.css}
├── requirements.txt
└── README.md
```

## Run it

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

python -m backend.ingest        # first-time data load (downloads trades + prices)
flask --app app run             # open http://127.0.0.1:5000
```

After the first load, click **⟳ Refresh data** in the UI (or re-run
`python -m backend.ingest`) to pull new purchases and update current prices.

### Useful env vars

| Var | Default | Purpose |
|-----|---------|---------|
| `CT_LOOKBACK_MONTHS` | `6` | how far back to include purchases |
| `CT_DB_PATH` | `data/trades.sqlite` | SQLite location |
| `CT_MAX_TICKERS` | `0` (all) | cap tickers priced per refresh (yfinance is slow) — handy for a quick demo |
| `CT_HOUSE_URL` / `CT_SENATE_URL` | public datasets | override the disclosure sources |

## Notes & limitations

- **Values are estimates.** Disclosures report a **dollar range** (e.g.
  `$1,001–$15,000`), not share counts. "Value at purchase" uses the range midpoint;
  "value now" = midpoint × (current price ÷ purchase price). Both are labeled as
  estimates in the UI.
- **"Current members":** the app shows members who *disclosed* purchases in the
  window. A strict "currently in office" filter would cross-reference a live
  roster (e.g. the `unitedstates/congress-legislators` dataset) — a reasonable
  future enhancement.
- **`yfinance` can be slow / rate-limited**; the first refresh prices many tickers
  (cached afterward). Use `CT_MAX_TICKERS` to keep a demo snappy.
- The SQLite file under `data/` is gitignored — it's regenerated by ingest.
